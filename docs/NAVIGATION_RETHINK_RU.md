# Переосмысление верхней навигации BookIndex

Дата: 2026-05-09

Цель: убрать плоскую "простыню" пунктов в шапке и разнести интерфейс по задачам пользователя, а не по внутренним типам данных.

## 1) Текущая проблема

Сейчас рядом оказываются разные уровни смысла:

1. Крупные разделы: `Материалы`, `Профессиональный аппарат`, `Корпус`.
2. Указатели: `Сводный указатель`, `Предметный`, `Имена`, `Топонимы`, `Этнонимы`, `Языки`, `Лексика`, `Лексика (обратная)`.
3. Инструменты: `KWIC`, `Глоссарий`, `Галерея лингвистов`, `Русский во времени`, `Фонетические законы`, `Проверьте себя`.
4. Режимы одного материала: `Лекции`, `Сравнение лекций`, `Страница лекции`, `Что почитать ещё`.

Из-за этого шапка выглядит как dump всех возможностей сразу: пользователь не понимает, где вход в чтение, где исследовательский аппарат, где справочник, а где инструменты.

## 2) Новый принцип

Первый уровень должен отвечать на вопрос "что я хочу сделать сейчас":

1. Читать и ориентироваться.
2. Найти сущность.
3. Исследовать связи.
4. Работать с корпусом.
5. Проверить себя.

Количество элементов и счетчики лучше показывать внутри разделов, а не в шапке. В шапке должны быть короткие смысловые входы.

## 3) Предлагаемый первый уровень

```text
Главная
Указатели
Материалы
Аппарат
Инструменты
Практикум
```

`Корпус` не нужен как пункт первого уровня. Он остается последним подпунктом в `Материалы`, потому что для пользователя это источник и база материалов, а не отдельный сценарий чтения.

## 4) Внутренняя структура

### Главная

Назначение: вход, объяснение сайта, быстрые маршруты.

Содержит:

1. Как пользоваться «Зализнякиадой».
2. Книга в цифрах.
3. Быстрые маршруты.
4. Недавние/рекомендуемые карточки.

### Указатели

Назначение: все списки сущностей, включая сводный.

Первый экран внутри раздела:

```text
Сводный указатель
Предметный
Имена
Топонимы
Этнонимы
Языки
Лексика
Лексика с конца
```

Правила:

1. `Сводный указатель` - default.
2. Счетчики показывать как вторичную мета-информацию: `3376`, `92`, `176`, но не делать их частью названия кнопки.
3. Внутри списка не показывать постоянные labels "из какого указателя" и "из какого источника"; это tooltip/hover.
4. Список должен быть читаемым в одну строку на элемент; длинное значение обрезается ellipsis, полный текст в hover.

### Материалы

Назначение: чтение и источники.

Содержит:

```text
Лекции
Страница лекции
Сравнение лекций
Что почитать ещё
Корпус
```

Правила:

1. `Лекции` - обзор всего лекционного материала.
2. `Страница лекции` - режим чтения текущей лекции/страницы.
3. `Сравнение лекций` - отдельный аналитический режим, но внутри материалов.
4. `Корпус` - последний пункт: источники, статусы, редакторские очереди.

### Аппарат

Назначение: профессиональный слой и научные визуализации.

Содержит:

```text
Профессиональный аппарат
Хронология открытий
Динамика по страницам
Визуализации
```

Правила:

1. Не дублировать хлебные крошки под этой строкой.
2. Визуализации не смешивать с обычными списками сущностей.
3. Внутренние anchors аппарата держать внутри страницы, а не выносить в шапку.

### Инструменты

Назначение: функции работы с текстом и справочником.

Содержит:

```text
Поиск
KWIC
Глоссарий
Галерея лингвистов
Русский во времени
Фонетические законы
```

Правила:

1. Глобальный поиск остается в шапке, но его результаты ведут в этот сценарий.
2. `KWIC` и `Глоссарий` должны быть рядом: оба являются инструментами чтения.
3. `Галерея`, `Русский во времени`, `Фонетические законы` лучше воспринимаются как исследовательские инструменты, а не материалы книги.

### Практикум

Назначение: активное взаимодействие.

Содержит:

```text
Проверьте себя
```

Пока это один пункт, но он заслуживает отдельного входа: это другой режим поведения, не чтение и не справочник.

## 5) Как это внедрять без резкой ломки

### Шаг 1

Оставить существующие route/hash без переименования. Поменять только группировку кнопок и заголовки.

### Шаг 2

Сделать первый уровень компактным:

```text
Главная | Указатели | Материалы | Аппарат | Инструменты | Практикум
```

Второй уровень показывать только внутри выбранного раздела.

### Шаг 3

Перенести счетчики из кнопок в заголовок или строку summary внутри раздела:

```text
Указатели: 3376 элементов
Имена 176 · Топонимы 98 · Этнонимы 66 · Языки 139 · Лексика 1367 · Лексика с конца 1403
```

### Шаг 4

Добавить `command palette`/универсальный поиск как главный способ перехода к редким инструментам. Чем меньше шапка, тем важнее хороший поиск.

## 6) Decision

Для ближайшей итерации не менять все маршруты. Сначала чинить читаемость и иерархию:

1. Сводный список - без узких колонок и постоянных labels.
2. Поиск - по всему доступному по умолчанию.
3. Переключатели поиска - только рядом с результатами.
4. Верхнее меню - готовить к модели `Главная / Указатели / Материалы / Аппарат / Инструменты / Практикум`.
