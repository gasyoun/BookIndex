# Переосмысление верхней навигации BookIndex

Дата: 2026-05-09

Цель: убрать плоскую "простыню" пунктов в шапке и разнести интерфейс по задачам пользователя, а не по внутренним типам данных.

## 1) Текущая проблема

Сейчас рядом оказываются разные уровни смысла:

1. Крупные разделы: `Материалы`, `Профессиональный аппарат`, `Корпус`.
2. Указатели: `Сводный указатель`, `Предметный`, `Имена`, `Топонимы`, `Этнонимы`, `Языки`, `Лексика`, `Лексика (обратная)`.
3. Инструменты: `KWIC`, `Глоссарий`, `Галерея лингвистов`, `Русский во времени`, `Фонетические законы`, `Проверьте себя`.
4. Режимы одного материала: `Лекции`, `Сравнение лекций`, `Страница лекции`, `Что почитать ещё`.

Из-за этого шапка выглядит как dump всех возможностей сразу: пользователь не понимает, где вход в чтение, где исследовательский аппарат, где справочник, а где инструменты.

## 2) Новый принцип

Первый уровень должен отвечать на вопрос "что я хочу сделать сейчас":

1. Читать и ориентироваться.
2. Найти сущность.
3. Исследовать связи.
4. Работать с корпусом.
5. Проверить себя.

Количество элементов и счетчики лучше показывать внутри разделов, а не в шапке. В шапке должны быть короткие смысловые входы.

## 3) Предлагаемый первый уровень

```text
Главная
Указатели
Материалы
Аппарат
Инструменты
Практикум
```

`Корпус` не нужен как пункт первого уровня. Он остается последним подпунктом в `Материалы`, потому что для пользователя это источник и база материалов, а не отдельный сценарий чтения.

## 4) Внутренняя структура

### Главная

Назначение: вход, объяснение сайта, быстрые маршруты.

Содержит:

1. Как пользоваться «Зализнякиадой».
2. Книга в цифрах.
3. Быстрые маршруты.
4. Недавние/рекомендуемые карточки.

### Указатели

Назначение: все списки сущностей, включая сводный.

Первый экран внутри раздела:

```text
Сводный указатель
Предметный
Имена
Топонимы
Этнонимы
Языки
Лексика
Лексика с конца
```

Правила:

1. `Сводный указатель` - default.
2. Счетчики показывать как вторичную мета-информацию: `3376`, `92`, `176`, но не делать их частью названия кнопки.
3. Внутри списка не показывать постоянные labels "из какого указателя" и "из какого источника"; это tooltip/hover.
4. Список должен быть читаемым в одну строку на элемент; длинное значение обрезается ellipsis, полный текст в hover.

### Материалы

Назначение: чтение и источники.

Содержит:

```text
Лекции
Страница лекции
Сравнение лекций
Что почитать ещё
Корпус
```

Правила:

1. `Лекции` - обзор всего лекционного материала.
2. `Страница лекции` - режим чтения текущей лекции/страницы.
3. `Сравнение лекций` - отдельный аналитический режим, но внутри материалов.
4. `Корпус` - последний пункт: источники, статусы, редакторские очереди.

### Аппарат

Назначение: профессиональный слой и научные визуализации.

Содержит:

```text
Профессиональный аппарат
Хронология открытий
Динамика по страницам
Визуализации
```

Правила:

1. Не дублировать хлебные крошки под этой строкой.
2. Визуализации не смешивать с обычными списками сущностей.
3. Внутренние anchors аппарата держать внутри страницы, а не выносить в шапку.

### Инструменты

Назначение: функции работы с текстом и справочником.

Содержит:

```text
Поиск
KWIC
Глоссарий
Галерея лингвистов
Русский во времени
Фонетические законы
```

Правила:

1. Глобальный поиск остается в шапке, но его результаты ведут в этот сценарий.
2. `KWIC` и `Глоссарий` должны быть рядом: оба являются инструментами чтения.
3. `Галерея`, `Русский во времени`, `Фонетические законы` лучше воспринимаются как исследовательские инструменты, а не материалы книги.

### Практикум

Назначение: активное взаимодействие.

Содержит:

```text
Проверьте себя
```

Пока это один пункт, но он заслуживает отдельного входа: это другой режим поведения, не чтение и не справочник.

## 5) Как это внедрять без резкой ломки

### Шаг 1

Оставить существующие route/hash без переименования. Поменять только группировку кнопок и заголовки.

### Шаг 2

Сделать первый уровень компактным:

```text
Главная | Указатели | Материалы | Аппарат | Инструменты | Практикум
```

Второй уровень показывать только внутри выбранного раздела.

### Шаг 3

Перенести счетчики из кнопок в заголовок или строку summary внутри раздела:

```text
Указатели: 3376 элементов
Имена 176 · Топонимы 98 · Этнонимы 66 · Языки 139 · Лексика 1367 · Лексика с конца 1403
```

### Шаг 4

Добавить `command palette`/универсальный поиск как главный способ перехода к редким инструментам. Чем меньше шапка, тем важнее хороший поиск.

## 6) Decision

Для ближайшей итерации не менять все маршруты. Сначала чинить читаемость и иерархию:

1. Сводный список - без узких колонок и постоянных labels.
2. Поиск - по всему доступному по умолчанию.
3. Переключатели поиска - только рядом с результатами.
4. Верхнее меню - готовить к модели `Главная / Указатели / Материалы / Аппарат / Инструменты / Практикум`.

## 7) Роль index-страницы

`index.html` становится основным публичным входом в BookIndex: это спокойная страница знакомства с книгой, проектом и главными способами чтения. Ее задача - не заменить рабочий интерфейс, а объяснить, зачем нужен BookIndex, и провести пользователя в `aaz-index.html`.

`aaz-index.html#v4/home/home` остается основным рабочим интерфейсом приложения. Именно там живет новая иерархия `Главная / Указатели / Материалы / Аппарат / Инструменты / Практикум`, поиск, карточки, указатели, корпус, аппарат и инструменты.

Правила:

1. `index.html` - главная публичная вводная страница для знакомства с книгой и проектом.
2. Главная кнопка действия на `index.html` ведет в рабочий маршрут `aaz-index.html#v4/home/home`.
3. `index.html` не должен быть placeholder и не должен автоматически перебрасывать пользователя без объяснения.
4. `index.html` не дублирует весь SPA: только короткое введение, 3-6 главных маршрутов и понятная кнопка "Открыть BookIndex".
5. Все рабочие сценарии, поиск, указатели, материалы, аппарат, инструменты и практикум живут в `aaz-index.html`.
6. Проверка готовности: `https://gasyoun.github.io/BookIndex/index.html` самодостаточно объясняет проект, а `aaz-index.html#v4/home/home` открывает полноценный рабочий интерфейс.

## 8) Инварианты дальнейшей работы

Эти правила фиксируют не только желаемую архитектуру, но и уже принятые решения после первых внедрений. Их нельзя откатывать при следующих итерациях навигации.

1. Первый уровень рабочего интерфейса остается коротким: `Главная / Указатели / Материалы / Аппарат / Инструменты / Практикум`.
2. `Корпус` не возвращается в первый уровень. Он живет внутри `Материалы` как последний пункт и ведет на `#v4/corpus/sources`.
3. Второй ряд навигации показывается только если у активного раздела больше одного локального пункта. Для одиночных разделов вроде `Практикум` не нужен дублирующий ряд из одной кнопки.
4. Третий ряд `view-tabs` нужен только там, где у текущего указателя есть несколько режимов просмотра. `#v4/all/list` не должен получать лишнюю строку режимов.
5. Счетчики указателей не возвращаются в labels верхнего меню. Они показываются внутри `index-section-summary`, рядом с навигационными chips раздела `Указатели`.
6. Переключатель плотности может показывать русские labels (`плотно`, `чтение`, `исследование`), но внутренние значения `compact`, `reader`, `research` остаются стабильными.
7. Темная тема удалена из текущей UI-стратегии. Не добавлять обратно `theme-dark`, `theme-btn` и отдельные dark-ветки CSS/JS; допустим только cleanup старого сохраненного состояния.
8. Старые hash-route сохраняются. Реформа меняет группировку и визуальную иерархию, а не адреса рабочих экранов.

Минимальный smoke-набор для будущих правок навигации:

| Маршрут | Ожидание |
|---|---|
| `aaz-index.html#v4/home/home` | Нет дублирующего второго ряда и третьего ряда. |
| `aaz-index.html#v4/all/list` | Есть раздел `Указатели`, нет `view-tabs`, список не ломается узкими колонками. |
| `aaz-index.html#v4/names/list` | Есть summary-chips указателей и `view-tabs` для режимов текущего указателя. |
| `aaz-index.html#v4/materials/lectures` | `Материалы` содержит лекции и `Корпус`, но `Корпус` не в первом уровне. |
| `aaz-index.html#v4/corpus/sources` | Редакторские очереди и статусы остаются на русском. |
| `aaz-index.html#v4/scholar/viz/module/viz01?century=21` | Нет лишней строки breadcrumbs поверх локальной навигации аппарата. |
| `aaz-index.html#v4/materials/tasks` | Одиночный `Практикум` не дублирует кнопку вторым рядом. |

Проверять эти маршруты минимум на desktop и mobile viewport. Для UI-изменений результат считается готовым только если нет горизонтального overflow, наложений header/search и возврата удаленных элементов.

Автоматизированная проверка этого контракта: `npm run check:navigation`. Она проходит по smoke-набору на desktop/tablet/mobile и должна оставаться зеленой при дальнейших изменениях header, tabs, поиска и routing.
