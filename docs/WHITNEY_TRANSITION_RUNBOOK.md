# Runbook: Перенос архитектуры Zalizniakiada на новый корпус (напр. Корни Whitney)

Этот документ описывает, как использовать наработки проекта Zalizniakiada v17.6 для создания аналогичной системы вокруг другого лингвистического материала (например, санскритских корней Уитни).

---

## 1. Ядро данных: Schema.org для лингвистики
Основа системы — файл `app_data.json`. Для переноса на Whitney вам нужно адаптировать структуру:
- **Lexicon (Лексикон)**: Здесь будут сами корни (dhatu).
- **Subject Index**: Грамматические категории (классы глаголов, типы основ).
- **Languages**: Родственные формы в других ИЕ языках.
- **Names**: Лингвисты (Панини, Уитни, Рену).

**Что забрать**: Логику `migrateAppDataSchema` из `core/data.js` для первичной обработки JSON.

---

## 2. Поисковый движок (Intellectual Search)
Движок Zalizniakiada поддерживает:
- **Deep Stemming**: Поиск не по точному совпадению, а по основе. Для санскрита это критично.
- **Web Worker**: Поиск не тормозит интерфейс даже при 10,000+ записей.

**Что забрать**: Файлы `src/core/search.js` и `src/core/search-worker.js`. Вам нужно будет только заменить правила стемминга в `utils/linguistics.js` на санскритские (или использовать упрощенные).

---

## 3. Модульный конструктор (Build System)
Вам не нужны тяжелые фреймворки (React/Vue).
- **bundle.js**: Наш самописный сборщик, который склеивает модули в один `v3_app.js`.

**Что забрать**: Папку `scripts/` и структуру `src/`. Это позволит вам разрабатывать чистый JavaScript, который работает везде.

---

## 4. Слой Digital Humanities (Аналитика)
Если у Whitney есть тысячи связей между корнями и производными словами:
- **Topic Clustering**: Модуль `core/analytics.js` автоматически найдет группы корней с похожими значениями.
- **Knowledge Hubs**: Вычислит самые «влиятельные» корни, через которые объясняется больше всего слов.

**Что забрать**: Логику `buildTopicClusters` и `calculateCentrality`.

---

## 5. Игровой движок (Gamification)
Любой академический материал сух. Мы оживили его через:
- **Quiz Engine**: `core/quiz.js`. Просто замените вопросы на задания по санскриту (например, определить класс корня).
- **Achievements**: `core/achievements.js`. Настройте значки за изучение определенных групп корней.

---

## 6. AI Insights (Интеллектуальные подсказки)
Наш «AI Copilot» — это не просто чат-бот, а эвристический алгоритм, который ищет пересечения в данных.
- **Что забрать**: `src/core/ai.js`. Он может автоматически подсказывать пользователю: «Этот корень Whitney часто встречается вместе с таким-то префиксом».

---

## 7. UI-фреймворк: State-driven rendering
Система построена на роутинге через хэши (`#v4/lexicon/list/item/...`).
- **Cards & Lists**: Модули `renderers/cards.js` и `renderers/lists.js` позволяют быстро создавать карточки для новых типов данных.

---

## Пошаговый план перехода (Whitney Edition):

1. **Данные**: Сформировать `app_data.json` с корнями Уитни в разделе `lexicon`.
2. **Сборка**: Скопировать `scripts/bundle.js` и настроить порядок файлов.
3. **Роутинг**: В `core/router.js` прописать новые маршруты (напр. `#v1/roots/list`).
4. **Лингвистика**: В `utils/linguistics.js` добавить поддержку транслитерации IAST/Devanagari (если нужно).
5. **Тесты**: Написать первый уровень квиза по основам Whitney в `core/quiz.js`.

**Архитектура Zalizniakiada v17.6 — это универсальный «движок для цифровых гуманитарных наук», который легко превращается в любой лингвистический атлас.**
