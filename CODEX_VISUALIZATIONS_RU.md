# Визуализации BookIndex — Регламент Codex

Цель: зафиксировать архитектуру, порядок реализации и критерии готовности для семи новых визуализационных модулей приложения.

---

## 1) Общие принципы

1. Каждый модуль — отдельный **lazy-loaded** JS-файл, подключаемый только при активации вкладки `«Визуализации»`.
2. Данные берутся **исключительно** из уже загруженного `app_data.json`; никаких дополнительных сетевых запросов.
3. При первом запуске модуля строится вспомогательный индекс (матрица совместных упоминаний, частотный словарь, хронотаблица) и кешируется в памяти (`window.__vizCache`).
4. Все модули обязаны корректно работать в **офлайн-режиме** (PWA, Service Worker уже есть).
5. Стилизация — через существующие CSS-переменные приложения; никаких внешних тем.
6. Каждый модуль регистрирует обработчик `visibilitychange` и **останавливает** анимации/таймеры при скрытии вкладки.

## 2) Модуль VIZ-01 — Слой-слайдер по векам (карта)

### Описание
Интерактивная карта Leaflet.js с временны́м слайдером. Отображает только сущности (учёные, памятники, диалектные ареалы), чья датировка попадает в выбранный диапазон веков.

### Стек
- `leaflet` (уже в `vendor/`)
- `leaflet-slider` или нативный `<input type="range">`
- Данные: поля `century`, `geo` из `app_data.json`

### Структура файла
```
scripts/viz/map-timeline.js
```

### Критерии готовности
- [ ] Слайдер отображает диапазон VIII–XXI вв.; шаг — 1 век.
- [ ] Маркеры перерисовываются без полной перезагрузки карты (`L.LayerGroup.clearLayers()`).
- [ ] Клик по маркеру открывает карточку сущности (тот же механизм, что в основном поиске).
- [ ] На мобильных (≤ 768 px) слайдер отображается снизу, карта занимает полную высоту.
- [ ] Офлайн: карта грузится из кеша тайлов (или показывает fallback-сообщение).

### Коммит-prefix
`feat(viz): map-timeline`

## 3) Модуль VIZ-02 — Граф сосуществования по лекциям

### Описание
Граф, где **узлы** — имена учёных и языки, **рёбра** — совместное упоминание на одной странице внутри одной лекции. Толщина ребра = количество совместных страниц.

### Стек
- `cytoscape` + лейаут `fcose` (быстрый для > 200 узлов)
- Данные: индекс `{lectureId → [{page, names, langs}]}`

### Структура файла
```
scripts/viz/cooccurrence-graph.js
```

### Алгоритм построения индекса
```js
// Псевдокод — реализовать в __vizCache.buildCoGraph()
for (const lecture of appData.lectures) {
  for (const page of lecture.pages) {
    const entities = [...page.names, ...page.langs];
    for (let i = 0; i < entities.length; i++)
      for (let j = i + 1; j < entities.length; j++)
        edge(entities[i], entities[j]).weight++;
  }
}
```

### Критерии готовности
- [ ] Граф строится за < 2 с на наборе из 2 МБ `app_data.json`.
- [ ] Узлы окрашены по типу: имя → `--color-primary`, язык → `--color-gold`.
- [ ] Фильтр по лекции (выпадающий список) перестраивает граф без мерцания.
- [ ] Двойной клик по узлу → переход к карточке сущности.
- [ ] Минимальный порог веса ребра настраивается ползунком (убирает шум).

### Коммит-prefix
`feat(viz): cooccurrence-graph`

## 4) Модуль VIZ-03 — Лента открытий (timeline по школам)

### Описание
Вертикальная хронологическая лента. Каждая карточка — учёный или термин с датой введения в научный оборот. Цветовая кодировка по научной школе. Фильтр-панель слева по чекбоксам.

### Стек
- Чистый CSS + JS (без внешних зависимостей)
- Данные: поле `school` + `year` из `app_data.json`

### Цветовая схема школ
| Школа | CSS-переменная |
|---|---|
| Сравнительно-историческая | `--color-primary` |
| Акцентология | `--color-orange` |
| Диалектология | `--color-success` |
| Прочие | `--color-text-muted` |

### Структура файла
```
scripts/viz/discovery-timeline.js
```

### Критерии готовности
- [ ] Лента рендерится в порядке убывания дат (новейшие сверху).
- [ ] Чекбоксы фильтруют мгновенно (CSS `display:none` без перестройки DOM).
- [ ] При клике на карточку → плавный скролл к ней + открытие карточки сущности.
- [ ] Пустое состояние при всех снятых фильтрах — отдельный дизайн (не пустой экран).

### Коммит-prefix
`feat(viz): discovery-timeline`

## 5) Модуль VIZ-04 — Тепловая матрица «лекция × термин»

### Описание
D3.js heatmap. Ось Y — лекции, ось X — термины (топ-50 по суммарной частоте). Интенсивность цвета = нормализованная частота упоминания. Кластеризация по обеим осям методом иерархической агломерации.

### Стек
- `d3` v7 (из `vendor/` или CDN с integrity-hash)
- `ml-hclust` для кластеризации (< 15 КБ, MIT)
- Данные: частотный словарь `__vizCache.termFreq[lectureId][term]`

### Структура файла
```
scripts/viz/heatmap-matrix.js
```

### Критерии готовности
- [ ] Матрица перерисовывается при изменении числа отображаемых терминов (слайдер top-N: 20/50/100).
- [ ] Дендрограммы по обеим осям отображаются рядом с матрицей.
- [ ] Ячейка при наведении показывает tooltip: `Лекция N · Термин · Частота: X`.
- [ ] Клик по ячейке → открывает страницу с совпадением в `aaz-index.html`.
- [ ] Экспорт матрицы в SVG через кнопку «Скачать».

### Коммит-prefix
`feat(viz): heatmap-matrix`

## 6) Модуль VIZ-05 — Сюжетная траектория «вопрос → аргумент»

### Описание
Sankey-диаграмма для структурированных лекций и раздела о «Слове о полку Игореве». Потоки: `Открытый вопрос → Гипотезы/Источники → Аргументы → Вывод`. Ширина потока = количество страниц/ссылок в соответствующем блоке.

### Стек
- `d3-sankey` (плагин D3)
- Данные: поле `narrative` из `app_data.json` (если отсутствует — ручная разметка в отдельном `narratives.json`)

### Структура файла
```
scripts/viz/narrative-sankey.js
narratives.json   ← данные разметки нарративов (если нужны)
```

### Критерии готовности
- [ ] Потоки анимированы при первом рендере (плавное появление, `duration: 600 мс`).
- [ ] Цвет узла = тематический блок (вопрос/источник/аргумент/вывод).
- [ ] Переключатель лекций (tab-bar над диаграммой).
- [ ] При отсутствии данных для лекции — информативный empty state.

### Коммит-prefix
`feat(viz): narrative-sankey`

## 7) Модуль VIZ-06 — Хорда связей между языками

### Описание
D3.js chord diagram. Показывает, какие языки чаще упоминаются совместно в одном контексте. Особенно ценен для компаративистики: выявляет «парные» языки и изолированные.

### Стек
- `d3` v7, `d3.chord()` + `d3.arc()`
- Данные: матрица `__vizCache.langCoMatrix`

### Структура файла
```
scripts/viz/lang-chord.js
```

### Критерии готовности
- [ ] При наведении на дугу — подсветка всех рёбер данного языка, остальные затухают до 10% opacity.
- [ ] Легенда с цветами языков кликабельна (toggle видимости).
- [ ] Фильтр: показывать только языки с частотой выше порога.

### Коммит-prefix
`feat(viz): lang-chord`

## 9) Общий препроцессор индексов (`__vizCache`)

Все модули зависят от одного препроцессора, который запускается **один раз** при инициализации вкладки «Визуализации».

```
scripts/viz/build-viz-cache.js
```

### Что строит
| Ключ кеша | Содержимое | Используется в |
|---|---|---|
| `termFreq` | `{lectureId: {term: count}}` | VIZ-04, VIZ-07 |
| `coGraph` | Список рёбер `{source, target, weight}` | VIZ-02 |
| `langCoMatrix` | Матрица N×N для языков | VIZ-06 |
| `termRankByLecture` | `{term: [rank_l1, rank_l2, ...]}` | VIZ-07 |
| `geoEntities` | `[{id, name, century, lat, lng}]` | VIZ-01 |

### Критерии готовности препроцессора
- [ ] Полный прогон на `app_data.json` завершается за < 3 с (Web Worker, чтобы не блокировать UI).
- [ ] Повторный вызов возвращает кеш мгновенно.
- [ ] При смене данных (reload) кеш сбрасывается.

## 11) Definition of Done для каждого VIZ-модуля

Модуль считается завершённым только если одновременно:

1. Файл `scripts/viz/viz-0N-*.js` запушен в `main`.
2. Модуль корректно рендерится в Chrome, Firefox, Safari (desktop + mobile).
3. Нет блокирующих ошибок в консоли (`console.error`).
4. Выполнены все чекбоксы из раздела «Критерии готовности» конкретного модуля.
5. Проверки из `CODEX_WORKFLOW_RU.md` раздела 5 пройдены.
6. Issue оформлен по `CODEX_WORKFLOW_RU.md` разделам 2 и 3.
7. Пользователю дана ссылка вида:
   `https://gasyoun.github.io/BookIndex/aaz-index.html?v=<short_sha>#viz`
