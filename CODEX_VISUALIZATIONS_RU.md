# Визуализации BookIndex — Регламент Codex v4.2

Цель: зафиксировать архитектуру, порядок реализации и критерии готовности для семи визуализационных модулей. Документ заменяет `CODEX_VISUALIZATIONS_RU.md` и включает правки спринта v4.2.

---

## 1) Общие принципы

1. Каждый модуль — отдельный lazy-loaded JS-файл, подключаемый только при активации соответствующего VIZ-таба.
2. Данные берутся исключительно из уже загруженного `app_data.json`; никаких дополнительных сетевых запросов.
3. При первом запуске модуля строится вспомогательный индекс и кешируется в памяти (`window.__vizCache`).
4. Все модули обязаны корректно работать в офлайн-режиме; это согласуется с уже добавленными PWA foundations и service worker.
5. Стилизация — через существующие CSS-переменные приложения; никаких внешних тем.
6. Каждый модуль регистрирует `visibilitychange` и останавливает анимации/таймеры при скрытии вкладки.
7. Каждый модуль со слайдером обязан уметь сериализовать состояние в URL и восстанавливать его при загрузке.

## 3) Сквозное состояние URL

В текущем проекте уже используется hash-routing с `HASH_ROUTE_PREFIX = 'v4'`, canonical hash и deep-link механика для карточек и scholar-навигации, поэтому состояние визуализаций нужно расширять в том же стиле, а не отдельным маршрутизатором. [cite:4]

### Новый helper

Файл:

```text
scripts/viz/viz-state.js
```

### Назначение

- хранить позицию слайдера;
- хранить активный фильтр;
- восстанавливать состояние модуля после reload;
- не ломать существующий `#v4/...`-маршрут.

### Формат

```text
#v4/scholar/viz/module/viz07?slider=5
#v4/scholar/viz/module/viz01?century=12&autoplay=0
```

### Псевдокод

```js
export function getVizHashBase() {
  return location.hash.split('?');
}

export function readVizParams() {
  const query = location.hash.split('?') ?? '';
  return new URLSearchParams(query);
}

export function writeVizParams(patch) {
  const base = getVizHashBase();
  const params = readVizParams();
  Object.entries(patch).forEach(([k, v]) => {
    if (v === null || v === undefined || v === '') params.delete(k);
    else params.set(k, String(v));
  });
  const next = params.toString();
  history.replaceState(null, '', next ? `${base}?${next}` : base);
}
```

### Критерии готовности

- [ ] После reload состояние слайдера восстанавливается.
- [ ] Копирование URL передаёт точную позицию.
- [ ] Параметры не ломают существующие deep-link сценарии.

## 4) VIZ-01 — Карта по векам

Текущий регламент описывает VIZ-01 как карту Leaflet с временным слайдером по векам и диапазоном VIII–XXI вв., поэтому новая задача — не переписывать модуль, а добавить autoplay-поведение поверх уже существующего слайдера. [cite:5]

### Изменение

Добавить кнопку `Play / Pause` рядом со слайдером.

### UX

- Нажатие `Play` запускает автоматическое продвижение на 1 век.
- Интервал по умолчанию: 800 мс.
- На последнем веке анимация останавливается.
- При ручном движении слайдера autoplay прекращается.
- При `visibilitychange` autoplay прекращается.

### Псевдокод

```js
let timer = null;

function stopAutoplay() {
  if (!timer) return;
  clearInterval(timer);
  timer = null;
  playBtn.setAttribute('aria-pressed', 'false');
  playBtn.textContent = '▶ Play';
}

function startAutoplay() {
  if (timer) return;
  playBtn.setAttribute('aria-pressed', 'true');
  playBtn.textContent = '⏸ Pause';
  timer = setInterval(() => {
    const current = Number(slider.value);
    const max = Number(slider.max);
    if (current >= max) {
      stopAutoplay();
      return;
    }
    slider.value = String(current + 1);
    renderCentury(current + 1);
    writeVizParams({ century: current + 1, autoplay: 1 });
  }, 800);
}
```

### Критерии готовности

- [ ] Есть кнопка `Play`.
- [ ] Есть состояние `Pause`.
- [ ] Анимация не создаёт второй параллельный таймер.
- [ ] При скрытии вкладки autoplay останавливается.
- [ ] Позиция слайдера пишется в URL.
- [ ] Карта по-прежнему не перезагружается полностью при смене века.

### Commit prefix

```text
feat(viz): add autoplay to map timeline
```

## 5) VIZ-02 — Граф сосуществования

Текущий регламент для VIZ-02 прямо предусматривает настраиваемый ползунок минимального веса ребра, но по твоему замечанию порог `20+` не встречается в реальных данных и только создаёт нефункциональный UI. [cite:5]

### Изменение

Удалить ползунок `min weight 20+` из UI и убрать связанную с ним ветку логики.

### Архитектурное решение

Вместо общего порога оставить только:
- фильтр по лекции;
- естественную толщину ребра по весу;
- при необходимости в будущем — компактный presets-фильтр (`all / 2+ / 3+`), но не сейчас.

### Псевдокод удаления

```js
// удалить:
renderMinWeightSlider();
state.minWeight = 20;

// заменить:
state.minWeight = 1;
```

### Критерии готовности

- [ ] В интерфейсе нет ползунка `min weight 20+`.
- [ ] Граф показывает реальные связи без пустого состояния из-за завышенного порога.
- [ ] Производительность не ухудшается.
- [ ] Фильтр по лекции работает как раньше.

### Commit prefix

```text
fix(viz): remove non-functional min-weight filter
```

## 6) VIZ-03 — Лента открытий

В текущем регламенте VIZ-03 задан как вертикальная лента с фильтр-панелью слева, но твоя правка указывает, что один жирный столбик тратит слишком много пустого места, поэтому архитектура должна перейти от “одна лента по центру” к компактной карточной сетке с сохранением хронологии. [cite:5]

### Изменение

Переверстать контейнер в responsive grid.

### Layout

- desktop: 2–4 колонки;
- tablet: 2 колонки;
- mobile: 1 колонка;
- карточки одинаковой визуальной плотности;
- сортировка по дате сохраняется.

### CSS-скелет

```css
.discovery-timeline {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(280px, 1fr));
  gap: 12px 14px;
  align-items: start;
}

.discovery-card {
  max-width: none;
  min-height: 100%;
}
```

### UX-правило

Фильтры не должны перестраивать весь DOM; допустимо скрытие карточек через класс:

```js
card.hidden = !matches;
```

### Критерии готовности

- [ ] На ширине desktop используется больше одной колонки.
- [ ] Нет широкого пустого поля справа.
- [ ] На mobile layout остаётся одноколоночным.
- [ ] Фильтрация не вызывает заметного “прыжка” интерфейса.

### Commit prefix

```text
feat(viz): compact multi-column discovery timeline
```

## 7) VIZ-07 — Bump-chart рангов

Текущий регламент VIZ-07 уже определяет график рангов по лекциям и хранение данных в `__vizCache.termRankByLecture`, а теперь к нему нужно добавить две вещи: deep-link на конкретную позицию слайдера и защиту подписей от наслоения и выхода за границы графика. [cite:5]

### Изменение A — deep-link состояния

Состояние выбранной позиции передавать в URL:

```text
#v4/scholar/viz/module/viz07?slider=5
```

Если модуль использует диапазон, формат допускает:

```text
#v4/scholar/viz/module/viz07?from=3&to=8
```

### Изменение B — защита подписей

Проблемы:
- текст наслаивается друг на друга;
- длинные слова вроде `калька`, `калькирован` могут вылезать вверх или наружу;
- подпись становится нечитаемой.

### Архитектурное решение

1. Увеличить верхний и правый margin SVG.
2. В обычном состоянии показывать подписи только для ограниченного набора линий, например top-5 или hovered.
3. Остальные подписи показывать только на hover/focus.
4. Добавить `clipPath` для области графика.
5. Для длинных слов — `truncateLabel()` с многоточием или перенос на `tspan`.

### Псевдокод

```js
const margin = { top: 28, right: 140, bottom: 24, left: 56 };

function truncateLabel(text, max = 14) {
  return text.length > max ? `${text.slice(0, max - 1)}…` : text;
}

function updateLabels(activeId = null) {
  labels.each(function(d, i) {
    const show = activeId ? d.id === activeId : i < 5;
    this.style.display = show ? '' : 'none';
    this.textContent = truncateLabel(d.label);
  });
}
```

### Дополнительное правило

Подпись не должна рисоваться за пределами `viewBox`; если расчётная точка вышла за границу, координата зажимается:

```js
labelY = Math.max(margin.top, Math.min(height - margin.bottom, labelY));
```

### Критерии готовности

- [ ] Ссылка может передавать точную позицию слайдера.
- [ ] Reload сохраняет выбранную позицию.
- [ ] Подписи не наслаиваются в default state.
- [ ] Длинные слова не выходят за пределы SVG.
- [ ] Hover показывает полную подпись активного термина.
- [ ] Текст читаем в desktop и mobile.

### Commit prefix

```text
fix(viz): bump-chart deeplink and label collision handling
```
