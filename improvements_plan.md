# Roadmap по улучшению проекта BookIndex

Этот документ содержит технические рекомендации по развитию архитектуры проекта «Зализнякиада». Рекомендации направлены на повышение стабильности, удобства разработки и автономности приложения.

## 1. Оптимизация инъекции данных (Data Injection)
Текущий метод через строковую замену в Python (`.replace()`) подвержен ошибкам экранирования.
- **Задача:** Перейти на использование тега `<script type="application/json">`.
- **Реализация:**
    1. В шаблоне `v3_template.html` разместить плейсхолдер внутри тега скрипта.
    2. В `v3_app.js` читать данные через `JSON.parse(document.getElementById('id').textContent)`.
- **Плюс:** Исключает "битую" кодировку и ошибки синтаксиса при сборке.

## 2. Современный стек сборки (Vite)
Замена самописного Python-скрипта сборки на стандарт индустрии.
- **Задача:** Внедрить **Vite** с плагином `vite-plugin-singlefile`.
- **Преимущества:**
    - Автоматическая минификация JS/CSS (уменьшение веса финального HTML).
    - Hot Module Replacement (HMR) — мгновенное обновление страницы при разработке.
    - Встроенная поддержка TypeScript или JSDoc.

## 3. Полноценный Offline-режим (PWA)
Сейчас карта и внешние изображения требуют интернета.
- **Задача:** Добавить Service Worker для кэширования тайлов Leaflet и медиа-ресурсов.
- **Результат:** Справочник станет на 100% автономным после первого открытия.

## 4. Модульность и масштабируемость данных
- **Разделение JSON:** Разбить монолитный `app_data.json` на логические части (карты, фонетика, метаданные) в репозитории. Склеивать их только на этапе сборки.
- **JSON Schema:** Создать схему для валидации данных, чтобы предотвратить структурные ошибки при добавлении новых материалов из книг.

## 5. Переход на декларативный UI (Alpine.js или Svelte)
Управление DOM через Vanilla JS становится сложным при росте количества фильтров и функций поиска.
- **Рекомендация:** Использовать **Svelte**.
- **Почему:** Он компилируется в чистый JS без рантайма, идеально подходит для создания "Single-File" приложений и значительно упрощает логику поиска (Fuse.js) и фильтрации.

## 6. Типизация
- **Задача:** Добавить JSDoc-комментарии или перейти на TypeScript.
- **Плюс:** Автодополнение в IDE для структуры данных `app_data`, что критично при работе с лингвистическими массивами данных.
