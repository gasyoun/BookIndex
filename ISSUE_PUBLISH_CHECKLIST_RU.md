# Чеклист публикации Issues (анти-мусор)

## Цель
Этот чеклист обязателен перед созданием/закрытием issue, чтобы не появлялись артефакты вида `????` и битая кириллица.

## Шаги (обязательно по порядку)
1. Подготовить текст issue только в UTF-8.
2. Проверить структуру:
   - есть `Цель:`
   - есть `Критерии готовности:`
3. Опубликовать issue/комментарий через JSON c `ensure_ascii=True` (чтобы избежать локальных кодировочных сюрпризов shell).
4. Сразу после публикации прогнать автопроверку:
   - `python scripts/issue_quality_guard.py --repo gasyoun/BookIndex --issues <номер> --strict-template`
5. Если проверка не прошла:
   - не закрывать issue;
   - сразу переписать title/body/comment и повторить проверку.
6. Только после зелёной проверки:
   - оставлять закрывающий комментарий;
   - переводить issue в `closed`.

## Быстрая проверка последних задач
- Точечно: `python scripts/issue_quality_guard.py --repo gasyoun/BookIndex --issues 63 64 65 --strict-template`
- Репозиторий целиком: `python scripts/issue_quality_guard.py --repo gasyoun/BookIndex --state all`

## Запреты
- Нельзя закрывать issue без пост-проверки скриптом.
- Нельзя оставлять issue/комментарии, где встречается `???` или массовые `?` вместо кириллицы.
